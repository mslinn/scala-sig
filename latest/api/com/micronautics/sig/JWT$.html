  4.  def apply(key: PrivateKey, subject: Subject, algorithm: SignatureAlgorithm = SignatureAlgorithm.RS256, audience: Audience = Audience.empty, claims: Claims = new DefaultClaims, compact: Boolean = true, deflate: Boolean = false, expiry: Date = oneHourFromNow, issuer: Issuer = Issuer.empty, payload: Payload = new Payload("")): JWT

    Factory class for creating JWT instances

    Factory class for creating JWT instances

    key

    the PrivateKey to sign the JWT with.

    subject

    This claim (sub) is required; identifies the principal that is the subject of the JWT. See https://tools.ietf.org/html/rfc7519#section-4.1.2

    algorithm

    Defaults to RSASSA-PKCS-v1_5 using SHA-256.

    audience

    The "aud" (audience) claim identifies the recipients that the JWT is intended for. Defaults to not being specified.

    claims

    Can specify additional claims with this parameter. Overrides other values which might conflict.

    compact

    If true, JSON will have whitespace removed, otherwise the JSON will be pretty-printed.

    deflate

    Set true to use a CompressionCodec to compress a large body. The JJWT library will automatically decompress and parse the JWT.

    expiry

    This sets the exp (expiration time) claim, which identifies the expiration time on or after which the JWT must not be accepted for processing. Defaults to one hour after the token is created.

    issuer

    This specifies the iss (issuer) claim, which identifies the principal that is issuing the JWT.

    payload

    Defaults to not being present.
